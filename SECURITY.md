Security Policy
Supported Versions
Please ensure you are using the latest version of the repository when running the Autonomous SRE Arena. Security updates are applied primarily to the main branch.

Version	Supported
Main	:white_check_mark:
< 1.0	:x:
Reporting a Vulnerability
Security is a priority for the Autonomous SRE Arena, especially since it involves deploying AI agents with kubectl access to a Kubernetes cluster. If you discover a security vulnerability within this project (e.g., unintended privilege escalation, unhandled prompt injection in the agent scripts, or an exposure in the FastAPI dashboard), we encourage you to report it immediately.

Please do not report security vulnerabilities through public GitHub issues.

Instead, please report them by emailing the maintainer directly at: [Insert Your Email Address]

Please include the following information in your report:

A description of the vulnerability and its potential impact.
Steps to reproduce the vulnerability.
Any potential fixes or suggestions you might have.
What to expect
You should receive an acknowledgment of your report within 48 hours.
We will investigate the issue and determine if it is a valid vulnerability.
If it is valid, we will work on a fix and provide an estimated timeline for the patch.
Once the issue has been resolved, we will publish a security advisory and notify you.
Thank you for helping keep the open-source community secure!
